# Тестовое задание

Разработать 2 приложения на C++ под ОС Linux.

1. Первое приложение: Приложение с интерфейсом командной строки, которое ждет ввода строки в терминал, затем читает эту строку из до символа перевода строки, передает эту строку во второе приложение с помощью методов IPC, читает ответную строку и выводит ее в терминал.

2. Второе приложение: Многопоточное приложение, которое умеет принять строку от первого приложения, перевернуть ее («asd» → «dsa»), отправить обратно в первое приложение перевернутую строку, а также записать в файл (залогировать) факт приема и обработки сообщения (строка в лог файле должна содержать: локальное время, исходную строку, время на обработку строки).

3. Старт скрипт на bash, который бы запускал оба приложения, а также умел оба их «убить».

Общие требования к обоим приложениям: Должно собираться с помощью Cmake 3.25+ и gcc, запускаться на ОС Ubuntu 22.04.

---

# Сборка и запуск
 `./build.sh` - сборка исполняемых файлов  
 `./apps_control.sh start/stop/restart` - запуск/остановка/перезапуск приложений

# IPC FIFO  
Для межпроессорного взаимодействия выбрал именованный FIFO так как:  
   - через него могут обмениваться любые процессы, имеющие права доступа;
   - работать с ними можно как с файлами
   - Гарантированный порядок записи/чтения FIFO
   - Синхронизация - FIFO блокирует процесс если он читает пустой канал или пишет в полный, пока не будет что-то помещено в FIFO или из него вычитано;
